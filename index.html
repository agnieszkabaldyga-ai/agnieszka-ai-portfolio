English Class Revision Generator

Problem it solves:
Manual post-lesson revision design is time-intensive, difficult to standardise, and inconsistent across instructors. This prompt systematises and automates personalised revision page generation while maintaining pedagogical control and brand consistency.

Use case:
Internal AI workflow implemented via Canva AI. Used after each lesson to generate individual, student-specific revision pages aligned strictly with lesson notes. Designed for multi-teacher deployment within the school.

Prompt architecture (constraint-based design & output control):

  • Output constrained to a responsive 1-page website (cross-device compatible)
  • Fixed brand elements (visual identity, school details, layout structure)
  • Variable placeholders (student name, lesson content) enabling controlled personalisation
  • Hard lexical constraint: vocabulary sourced exclusively from lesson notes
  • Difficulty level controlled implicitly via strict alignment with class notes
  • Structured exercise framework (gap-fill, matching, multiple choice)
  • Embedded output validation and scoring logic with cumulative percentage calculation
  • Reusable prompt architecture supporting multi-teacher deployment
  • Prompt standardisation ensuring consistent pedagogical outputs across instructors
  • Workflow optimisation reducing revision preparation time
  • Human-in-the-loop verification before publication to ensure quality control
Code a 1-page website with exercises for my student of English. It should work on computers, mobiles and tablets.
It should have  the name of my school with this emoji of a thread and a needle: 🪡Tailored Talk at the top.

The title of the website will be : ....... - revision from ...

It should be in my brand colours: navy dark blue, light powder pink and white. 
It should have some lively emojis connected wth studying, English and emojis referring to some words from the lesson if possible.  It can have some motivating comments like "Well done!" etc. 
At the bottom of the website there should be my e-mail: kontakt@tailoredtalk.eu and the website address of my school: www.tailiredtalk.eu 


Make 3 exercises
1. Fill in the gaps. There is a box with words from the lesson and below sentences in English with gaps. There should be more words than gaps - some words will not be used in any sentence. The student needs to write the words from the box into the sentences. The words in the box must have vocabulary only from the lesson notes, do not invent any words. 
Matching. Match words (1-5) with definitions (A-E). The student should write the correct letter next to the correct number (no dragging).
Multiple choice. The student needs to click on the right answer from a, b, and c.

After completing each exercise, student can click on “Check my score” button. Then the good answers become green and the wrong answers become red and there is a final mark visible. Every good answer is worth 1 point. At the end, add a final score summary section that calculates and displays the total points and percentage across all three exercises.

Group Revision/Test Platform Generator

Problem it solves:

Group revision requires individual accountability, structured result tracking, and teacher-level oversight. Manual preparation of interactive revision pages and tests for multiple students is time-intensive and difficult to standardise. This prompt extends the individual revision workflow to support controlled multi-student interaction and evaluation.

Use case:

Internal AI workflow implemented via Canva AI. Used to generate interactive revision pages and tests for group classes, expanding the individual system into a structured multi-user environment.

Prompt architecture (constraint-based design & output control):

  • Builds on all constraint mechanisms from Prompt 1 with extended multi-user logic
  • Multi-user handling with predefined student identifiers
  • Role-separated output structure (Student View / Teacher View)
  • Individual result tracking per student
  • Detailed answer breakdown visible in Teacher View
  • State control mechanism: answers locked after submission
  • Controlled visibility of results (students see only their own outcomes; teacher sees full class data)
