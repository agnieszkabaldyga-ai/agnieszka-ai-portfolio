English Class Revision Generator

Problem it solves:
Manual post-lesson revision design is time-intensive, difficult to standardise, and inconsistent across instructors. This prompt systematises and automates personalised revision page generation while maintaining pedagogical control and brand consistency.

Use case:
Internal AI workflow implemented via Canva AI. Used after each lesson to generate individual, student-specific revision pages aligned strictly with lesson notes. Designed for multi-teacher deployment within the school.

Prompt architecture (constraint-based design & output control):

  • Output constrained to a responsive 1-page website (cross-device compatible)
  • Fixed brand elements (visual identity, school details, layout structure)
  • Variable placeholders (student name, lesson content) enabling controlled personalisation
  • Hard lexical constraint: vocabulary sourced exclusively from lesson notes
  • Difficulty level controlled implicitly via strict alignment with class notes
  • Structured exercise framework (gap-fill, matching, multiple choice)
  • Embedded output validation and scoring logic with cumulative percentage calculation
  • Reusable prompt architecture supporting multi-teacher deployment
  • Prompt standardisation ensuring consistent pedagogical outputs across instructors
  • Workflow optimisation reducing revision preparation time
  • Human-in-the-loop verification before publication to ensure quality control

This prompt extends the individual revision workflow to support controlled multi-student interaction and evaluation.

Case Study



Case Study Summary

This project evaluates how three large language models — ChatGPT, Gemini, and Copilot — perform when executing a highly constrained creative-writing prompt combined with a structured language exercise task.

The experiment includes one prompt refinement which emerged during testing after observing weaknesses in the initial output.

The results reveal differences in constraint compliance, formatting reliability, and task execution across models.

Objective

To evaluate how three LLM systems, ChatGPT, Gemini and Copilot, perform when executing a highly constrained creative-writing and language exercise design prompt.

Evaluation Focus

Based on extensive personal experience using AI to generate multiple-choice exercises, and repeated manual refinement of narrative outputs for student use, this case study examines whether current models (2026) can reliably embed a language exercise within a coherent narrative, or whether expert human input remains necessary to ensure quality. Model performance was examined across the three output dimensions listed below.

• Output Format

Which model demonstrates the strongest compliance under stacked instructional constraints with respect to output format?

• Narrative Quality

Are the models able to maintain narrative logic, grammatical correctness, and humour under lexical restrictions?

• Language Exercise Design Accuracy

Are the models able to design a logically embedded multiple-choice task within the narrative that adheres to specified methodological requirements?

Prompt Design: Constraint Stacking Framework

  • Linguistic constraints: CEFR A2, Past Simple only.
  • Structural constraints: introduction–development–ending structure, sentence limit.
  • Lexical constraints: predefined vocabulary and linking words pool.
  • Formatting constraints: gap-fill exercise with options, 100% pool vocabulary coverage in answer options, explicit template for gaps and correct-answer annotation.
  • Creative constraint: humorous story, for a specific age, fixed characters, single-topic narrative, logical event progression.

Prompt 1

Prompt 1 was first tested with ChatGPT.

Write a logical and simple story in English at A2 level. 
The story must contain 10–20 sentences in total and use only the Past Simple tense. 
Sentences must be short and uncomplicated. 
The story must have one clear main plot and a visible three-part structure: Paragraph 1 – Introduction Paragraph 2 – Development Paragraph 3 – Ending. 

Events in the story must be logically connected. You may use only the following connectors: and, but, or, so, after that, when, that’s why, because. Do not use any other connectors. 

The story should be humorous and suitable for an English learner aged 30–45. The story must be about a woman named Malwina (the main character) who has two children: a teenage daughter and a 9-year-old son. 

Include fill-in-the-blank sentences with three answer options. Only one option must correctly complete the sentence, and the other two options must be either illogical or grammatically incorrect. 
Format sentences with gaps exactly like this example: “The police caught him and he went to ______. OPTIONS: a) jail b) jails c) flour (correct: a)” 

Important constraints: 
•	I will provide a list of words at the end of my message.
•	The answer options a), b), and c) must be created only using words from this list. 
•	Do not invent any new words for the options. 
•	Every word from the provided list must appear at least once among the answer options (a, b, or c). 
•	The correctly completed story must contain at least 75% of the words from the provided list. 
•	The entire text must be written only in English. 
•	Do not include Polish translations. 
•	Return the response as three paragraphs (Introduction, Development, Ending) with 10–20 sentences in total.

Key Takeaways

  • ChatGPT achieved the best results across all criteria.
  • Gemini performed well narratively but failed to comply with several formatting instructions
  • Copilot achieved the worst results across all criteria.

Why This Case Study Matters

This experiment demonstrates how prompt engineering and systematic evaluation can be used to assess the reliability of AI systems for structured educational content generation. The findings indicate that current models still require targeted training and human-guided refinement to produce high-quality narratives and effective humour, particularly in content designed for foreign language learning.

Possible Directions for Further Research

  1. Prompt language effects: How does prompt language (e.g., English vs Polish) influence model performance and instruction adherence?
  2. Prompt stability across content variations: Do results remain consistent when the same prompt structure is applied to different characters and vocabulary sets?
  3. Further prompt refinement: How does explicitly requiring gap-fill sentences to be embedded within the narrative affect model performance and scoring?
